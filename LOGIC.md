# Lost & Found Bot — Логика работы

## Общее описание

Telegram-бот для публикации объявлений о потерянных и найденных вещах. Пользователь проходит пошаговый диалог (ConversationHandler), по итогу которого запись сохраняется в CSV-файл.

---

## Стек

| Компонент | Технология |
|-----------|-----------|
| Бот | `python-telegram-bot` (async) |
| Хранение данных | CSV-файлы (`database.py`) |
| Конфиг | `.env` → `BOT_TOKEN` |

---

## Команды

| Команда | Действие |
|---------|----------|
| `/start` | Начать новый диалог — выбор «потерял / нашёл» |
| `/cancel` | Отменить текущий диалог на любом шаге |
| `/help` | Справка по работе бота |

---

## Диалог (ConversationHandler)

Бот ведёт пользователя через 5 последовательных состояний:

```
/start
  │
  ▼
┌─────────────────────┐
│ 0. CHOOSING_TYPE    │  Inline-кнопки: "I Lost Something" / "I Found Something"
└────────┬────────────┘
         │ callback_data = "lost" | "found"
         ▼
┌─────────────────────┐
│ 1. CATEGORY         │  Reply-клавиатура с категориями (14 шт.)
└────────┬────────────┘
         │ текст кнопки → маппинг в чистое название через CATEGORY_MAP
         ▼
┌─────────────────────┐
│ 2. DESCRIPTION      │  Свободный текст — описание предмета
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 3. PHOTO            │  Отправка фото или «Skip Photo»
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ 4. REWARD_OR_CONTACT│  Если lost → сумма вознаграждения или «No reward»
│                     │  Если found → контактные данные (телефон/email/username)
└────────┬────────────┘
         │
         ▼
    Сохранение в CSV
    Вывод сводки
    ConversationHandler.END
```

---

## Подробное описание каждого шага

### 0. CHOOSING_TYPE

- **Триггер:** `/start`
- **UI:** Inline-клавиатура с двумя кнопками
- **Данные:** `context.user_data['item_type']` = `"lost"` или `"found"`
- **Следующее состояние:** CATEGORY

### 1. CATEGORY

- **UI:** Reply-клавиатура 5 рядов (14 категорий с эмодзи)
- **Категории:** Toy, Phone, Laptop or Tablet, Bag or Bagpack, Wallet, Keys, Document, Clothing, Jewelry, Watches, Glasses or Sunglasses, Earphones or Earbuds, Pets, Other
- **Маппинг:** эмодзи-текст кнопки → чистое имя через `CATEGORY_MAP`
- **Данные:** `context.user_data['category']`
- **Следующее состояние:** DESCRIPTION

### 2. DESCRIPTION

- **UI:** свободный ввод текста
- **Подсказка:** цвет, размер, бренд, особые приметы
- **Данные:** `context.user_data['description']`
- **Следующее состояние:** PHOTO

### 3. PHOTO

- **UI:** ожидание фото или кнопка «Skip Photo»
- **Обработка:**
  - Фото получено → `photo_file_id = update.message.photo[-1].file_id`
  - Skip → `photo_file_id = None`
- **Следующее состояние:** REWARD_OR_CONTACT

### 4. REWARD_OR_CONTACT

- **Если `item_type == "lost"`:**
  - Запрос суммы вознаграждения или «No reward»
  - Данные: `context.user_data['reward']`
- **Если `item_type == "found"`:**
  - Запрос контактной информации (телефон, email, Telegram username)
  - Данные: `context.user_data['contact_info']`
- **Далее:** сохранение в CSV → сводка → END

## Хранение данных (CSV)

Данные хранятся в двух CSV-файлах. Файлы создаются автоматически при импорте `database.py`.

### Файл `lost_items.csv`

| Поле | Описание |
|------|----------|
| id | Автоинкремент |
| user_id | Telegram user ID |
| category | Категория предмета |
| description | Описание |
| photo_file_id | Telegram file_id фото (может быть пустым) |
| reward | Вознаграждение (может быть пустым) |
| created_at | ISO timestamp создания записи |

### Файл `found_items.csv`

| Поле | Описание |
|------|----------|
| id | Автоинкремент |
| user_id | Telegram user ID |
| category | Категория предмета |
| description | Описание |
| photo_file_id | Telegram file_id фото (может быть пустым) |
| contact_info | Контактные данные нашедшего (может быть пустым) |
| created_at | ISO timestamp создания записи |

### Функции (`database.py`)

| Функция | Назначение |
|---------|-----------|
| `init_database()` | Создаёт CSV-файлы с заголовками, если не существуют |
| `save_lost_item(...)` → `int` | Добавляет строку в `lost_items.csv`, возвращает ID |
| `save_found_item(...)` → `int` | Добавляет строку в `found_items.csv`, возвращает ID |
| `get_lost_items(limit)` → `list[dict]` | Последние N потерянных вещей |
| `get_found_items(limit)` → `list[dict]` | Последние N найденных вещей |

---

## Сводка (Summary)

После сохранения бот формирует сообщение-сводку, которое включает:

- Report ID (из CSV)
- Тип: Lost / Found
- Категория
- Описание (обрезается до 100 символов)
- Фото: есть / нет
- Reward (для lost) или Contact (для found, обрезается до 50 символов)

---

## Сопоставление предметов

Бот использует функцию `is_similar()` для автоматического поиска совпадений в обоих направлениях: после регистрации найденного предмета ищет похожие потерянные, и наоборот. Логика обоих потоков симметрична.

---

### A. Поток «Найденный предмет → поиск потерянных» (Found → Lost)

После того как пользователь (ЮН) регистрирует **найденный** предмет (НО), бот автоматически ищет похожие потерянные предметы (ППО) через `find_similar_lost_items()`.

#### Поток для ЮН (нашедшего)

```
Найденный предмет сохранён
  │
  ▼
find_similar_lost_items(НО) → ППО[]
  │
  ├─ ППО пусто → "Похожих не найдено", END
  │
  ▼
Показываем ППО[i] с кнопками: [Да] [Нет] [Не уверен]
  │
  ├─ "Нет" → i++ → показать следующий ППО (или END если закончились)
  │
  ├─ "Не уверен" (ППО_НУ) →
  │     • Отправить НО (без контактов) владельцу ППО_НУ
  │     • i++ → показать следующий ППО
  │
  └─ "Да" (ППО_ДА) →
        • found_items.csv: is_matched НО = id ППО_ДА
        • Отправить НО (без контактов) владельцу ППО_ДА
        • END (больше ППО не показываем)
```

#### Поток для владельца потерянного предмета (ответ на уведомление)

Когда владелец получает уведомление о НО, ему предлагаются кнопки: [Да] [Нет] [Не уверен]

```
Владелец получает НО (без контактов ЮН)
  │
  ├─ "Нет" → "Спасибо, продолжаем искать"
  │
  ├─ "Не уверен" → отправить контакты ЮН владельцу
  │
  └─ "Да" →
        • lost_items.csv: is_matched ППО = id НО
        • Отправить контакты ЮН владельцу
```

---

### B. Поток «Потерянный предмет → поиск найденных» (Lost → Found)

После того как пользователь (ЮП) регистрирует **потерянный** предмет (ПО), бот автоматически ищет похожие найденные предметы (ПНО) через `find_similar_found_items()`.

#### Поток для ЮП (потерявшего)

```
Потерянный предмет сохранён
  │
  ▼
find_similar_found_items(ПО) → ПНО[]
  │
  ├─ ПНО пусто → "Похожих не найдено", END
  │
  ▼
Показываем ПНО[i] с кнопками: [Да] [Нет] [Не уверен]
  │
  ├─ "Нет" → i++ → показать следующий ПНО (или END если закончились)
  │
  ├─ "Не уверен" (ПНО_НУ) →
  │     • Отправить ПО (без контактов) нашедшему ПНО_НУ
  │     • i++ → показать следующий ПНО
  │
  └─ "Да" (ПНО_ДА) →
        • lost_items.csv: is_matched ПО = id ПНО_ДА
        • Отправить ПО (без контактов) нашедшему ПНО_ДА
        • END (больше ПНО не показываем)
```

#### Поток для нашедшего (ответ на уведомление)

Когда нашедший получает уведомление о ПО, ему предлагаются кнопки: [Да] [Нет] [Не уверен]

```
Нашедший получает ПО (без контактов ЮП)
  │
  ├─ "Нет" → "Спасибо, проверили"
  │
  ├─ "Не уверен" → отправить контакты нашедшего юзеру ЮП
  │
  └─ "Да" →
        • found_items.csv: is_matched ПНО_ДА = id ПО
        • Отправить контакты нашедшего юзеру ЮП
```

### Обработчики

| Обработчик | Тип | Назначение |
|------------|-----|-----------|
| `start_found_matching` | Вспомогательная | Запуск поиска ППО после регистрации найденного предмета |
| `show_next_match` | Вспомогательная | Показ следующего ППО нашедшему |
| `handle_finder_match_response` | ConversationHandler (FOUND_MATCHING) | Обработка Да/Нет/Не уверен от ЮН |
| `notify_lost_item_owner` | Вспомогательная | Отправка НО владельцу потерянного предмета |
| `handle_owner_response` | Standalone CallbackQueryHandler (group 1) | Обработка Да/Нет/Не уверен от владельца ППО |
| `start_lost_matching` | Вспомогательная | Запуск поиска ПНО после регистрации потерянного предмета |
| `show_next_lost_match` | Вспомогательная | Показ следующего ПНО потерявшему |
| `handle_loser_match_response` | ConversationHandler (LOST_MATCHING) | Обработка Да/Нет/Не уверен от ЮП |
| `notify_found_item_owner` | Вспомогательная | Отправка ПО нашедшему (владельцу found item) |
| `handle_found_owner_response` | Standalone CallbackQueryHandler (group 1) | Обработка Да/Нет/Не уверен от нашедшего ПНО |

### Callback data

| Паттерн | Источник | Направление |
|---------|----------|-------------|
| `finder_yes_{lost_id}` / `finder_no_{lost_id}` / `finder_unsure_{lost_id}` | Кнопки ЮН (нашедшего) | Found → Lost |
| `owner_yes_F{found_id}_L{lost_id}` / `owner_no_...` / `owner_unsure_...` | Кнопки владельца потерянного | Found → Lost |
| `loser_yes_{found_id}` / `loser_no_{found_id}` / `loser_unsure_{found_id}` | Кнопки ЮП (потерявшего) | Lost → Found |
| `fowner_yes_L{lost_id}_F{found_id}` / `fowner_no_...` / `fowner_unsure_...` | Кнопки нашедшего (уведомление) | Lost → Found |

## Функции сопоставления (`database.py`)

| Функция | Назначение |
|---------|-----------|
| `is_similar(lost, found)` → `bool` | Сравнение предметов (сейчас — по категории) |
| `get_unmatched_lost_items()` → `list[dict]` | Все потерянные предметы с `is_matched == "false"` |
| `get_unmatched_found_items()` → `list[dict]` | Все найденные предметы с `is_matched == "false"` |
| `find_similar_lost_items(found)` → `list[dict]` | Найти ППО для данного найденного предмета |
| `find_similar_found_items(lost)` → `list[dict]` | Найти ПНО для данного потерянного предмета |
| `get_lost_item_by_id(id)` → `dict \| None` | Получить потерянный предмет по ID |
| `get_found_item_by_id(id)` → `dict \| None` | Получить найденный предмет по ID |
| `update_lost_item_match(lost_id, found_id)` | Обновить `is_matched` в `lost_items.csv` |
| `update_found_item_match(found_id, lost_id)` | Обновить `is_matched` в `found_items.csv` |

## Точка входа

```
main() →
  1. Читает BOT_TOKEN из .env
  2. Создаёт Application
  3. Регистрирует ConversationHandler (entry: /start, fallback: /cancel)
  4. Регистрирует /help
  5. Запускает polling
```
